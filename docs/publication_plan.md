# Publication Plan

## Public GitHub Repository

Commit the public repository package generated by:

```bash
scripts/build_publication.sh
```

The public repository should include:

- source code
- tests
- public corpus files
- model result JSONL files
- Markdown reports
- documentation
- license and citation metadata

The public repository should not include:

- `.env`
- API keys
- virtual environments
- private corpora
- answer-only files
- exploratory canary runs

## Scientific Supplement

Attach the frozen release archive as a supplement:

```text
liveproof-rc-v1-frontier-v2.tar.gz
```

This archive contains the private corpora needed to audit every score. Once it
is public, it should not be reused as a hidden leaderboard.

## Suggested Title

```text
Static Benchmarks Are Dead:
Verifier-Backed Post-Training Evals Reveal Structured Algorithmic Blind Spots
```

## Narrow Public Claim

```text
LiveProof shows that fresh seed-generated tasks with exact verifiers can expose
structured algorithmic blind spots that aggregate static benchmarks hide.
```

## Claims to Avoid

- Do not claim LiveProof is a complete AGI benchmark.
- Do not claim the initial model study is a statistically definitive leaderboard.
- Do not claim private corpus results remain hidden after publication.
- Do not present estimated GPT or Claude scores as measured results.

## Launch Sequence

1. Create a GitHub repository from the public package.
2. Upload the supplement archive and checksum as a GitHub Release.
3. Mint an archival DOI with Zenodo or an equivalent artifact repository.
4. Submit the preprint text after replacing placeholder repository and DOI links.
5. Publish a short technical announcement with the corpus hashes and measured
   DeepSeek V4 Pro result.
